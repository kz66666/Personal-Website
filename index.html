Dynamic Memory Allocation

  • Used an array of free lists each segregated by size class and organized as a circular, doubly linked list
  • Implemented additional quick lists for recently freed small blocks, organized as singly linked list
  • Each block has a header and, if free, a footer, which inclues information on the current block size, if it is allocated, if the previous block is allocated, and is it in the quick list. The header and footer are used to simplify coalescing free blocks to minimize fragmentation
